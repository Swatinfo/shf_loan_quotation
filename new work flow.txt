  ---
  11-Stage Workflow — Role by Role

  Stage 1: Inquiry

  - Auto-completed on quotation conversion (no manual action)

  Stage 2: Document Selection

  - Auto-completed on quotation conversion (no manual action)

  Stage 3: Document Collection

  - Loan Advisor — Collects all required documents, marks them Received/Rejected/Waived, uploads attachments

  Stage 4: Parallel Processing (5 sub-stages)

  ---
  Auto-Assignment Priority

  When a stage needs an assignee, the system checks in order:
  1. Product Stage Config (branch/city/state-specific user)
  2. Bank's default employee (for bank_employee stages)
  3. Bank employee matching the branch
  4. Bank employee (any branch)
  5. Default office employee for branch (for office_employee stages)
  6. Loan creator (if advisor/branch_manager)
  7. Any user with eligible role in same branch
  8. Any active user with eligible role

  ---
  Visibility Rules

  A user can see a loan if they:
  - Have view_all_loans permission, OR
  - Created the loan, OR
  - Are the assigned advisor, OR
  - Are assigned to any stage, OR
  - Are a branch manager for the loan's branch
